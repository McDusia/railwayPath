About

The application delivers high quality GIS solutions for customers involved in process of railway paths investment. Our application supports process of choosing railway path. User can draw a line as a new railway path and gets the information about prices of areas needed to build such path. Prices are estimated using machine learning. What is more some path validations are implemented such as checking if the proposed path does not exceed the minimal radius of curves and maximal fall in area.

Application features:
  • Possibility to draw path on real very accurate map
  • Displaying parcels from selected area
  • Displaying some information about parcels (including predicted price)
  • Trace validation - checkoing if minimal radius of curvvature is not exceded or maximal fall in the area are not exceeded
  • Prediction of sumarized trace price

Help

Below are presented parameters, which can be adjusted to needs. Based on them application will validate your trace.

  1. Design speed [km/h]

    2. The speed, which ensure save ride for one vehicle in normal condition. This parameter has to be established at the beginning of planning train path. It depends on path function and kind of transport.

  2. Minimum radius of curvature [m]

    3. For specified design speed, trace must not exceed minimum radius of curvature. It is possibility to add more restriction for curvature. Adjust this parameter to your needs.

  3. Maximal fall in the area [%]

    4. For specified design speed, trace must not exceed maximal fall in the area. The trace must not have too huge grade for specified type of rail vehicle. This factor is very important. Adjust it.

  4. Width of the road under the railway [m]

    5. This parameter specified how width trace application will draw on the map. You can also draw on your own extra parcels for traction power infrastructure, maintenance facility, railway stations e.t.c.
